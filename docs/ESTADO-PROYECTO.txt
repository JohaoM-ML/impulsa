--------------------------------------------------------------------------------
1. RESUMEN
--------------------------------------------------------------------------------
PWA mobile-first para bodegas / pymes informales del Perú.
Objetivo: que el dueño OPERE (registrar ventas, inventario, gastos),
ENTIENDA (dashboard y PymScore en su propio lenguaje) y CREZCA
(educacion financiera + acceso a credito).

Stack: Next.js 14 (App Router) + TypeScript estricto + Supabase
(PostgreSQL + Auth + RLS) + Claude (claude-sonnet-4-6) + OpenAI Whisper
 + [n8n (WhatsApp, solo orquestacion) + Vercel.] esto ultimo aun no esta hecho

Estado general: FUNCIONAL EN LOCAL. Build y lint en verde.
El servidor de desarrollo levanta en http://localhost:3000

3. NAVEGACION (5 secciones del nav inferior)
--------------------------------------------------------------------------------
/inicio       Dashboard del dia: resumen, PymScore, accesos rapidos
/registrar    Tabs Foto / Voz / Manual + toggle Venta/Compra
/mi-negocio   Tabs Flujo / Inventario / Top / Fiado
/pym-score    Score, componentes y evolucion (nivel 2+)
/aprender     Modulos por nivel + examen

Auth:        /login, /registro
Onboarding:  /onboarding (quiz + agente Claude que asigna nivel/XP)
Raiz /:      redirige a /login o /inicio segun sesion

Rutas legacy (redirigen automaticamente, codigo en app/(dashboard)/_legacy/):
  /ventas, /ventas/nueva       -> /registrar
  /inventario, /nueva-guia     -> /mi-negocio | /registrar
  /gastos                      -> /registrar
  /clientes                    -> /mi-negocio

4. API ROUTES (21 rutas)
--------------------------------------------------------------------------------
Negocio/dashboard:  /api/negocio  /api/dashboard  /api/seed
Operaciones:        /api/ventas  /api/inventario  /api/inventario/bulk
                    /api/gastos  /api/clientes  /api/proveedores
                    /api/mi-negocio/flujo  /api/mi-negocio/top
IA (server-side):   /api/ia/extraer-foto  /api/ia/transcribir
                    /api/ia/estructurar  /api/ocr
Niveles/score:      /api/nivel  /api/onboarding  /api/modulos
                    /api/examen  /api/pym-score
Chatbot:            /api/chatbot  (STUB funcional, ver seccion 6)

Regla: todas (excepto /api/chatbot) usan sesion Supabase y sacan negocio_id
de la sesion, NUNCA del body.
