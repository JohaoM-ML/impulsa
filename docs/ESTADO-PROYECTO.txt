1. RESUMEN
--------------------------------------------------------------------------------
Impulsa es una PWA mobile-first para micro y pequeñas empresas peruanas
informales, pensada especialmente para bodegas, puestos, negocios familiares y
emprendimientos que hoy registran sus ventas "en la cabeza", en cuadernos o por
WhatsApp. El usuario objetivo no es un contador ni un usuario avanzado de
Excel: es un dueño de negocio que necesita respuestas simples, rapidas y en su
propio lenguaje.

La tesis del producto es que la formalizacion financiera no empieza con un
software contable, sino con capturar el desorden diario del negocio: ventas,
compras, gastos, fiados, inventario y conversaciones. A partir de ese historial,
Impulsa ayuda al dueño a:

  OPERAR    Registrar ventas, compras, inventario, gastos y fiados sin friccion.
  ENTENDER  Ver su negocio con dashboard, PymScore y vocabulario adaptado.
  CRECER    Aprender finanzas por nivel y construir evidencia para credito.

El diferencial central no es solo "registrar datos", sino convertir esos datos
en una narrativa financiera accionable. Por eso el producto combina una app PWA
con un asesor por WhatsApp: el dueño puede registrar desde la interfaz visual o
escribirle al bot como le escribiria a una persona.

Stack: Next.js 14 (App Router) + TypeScript estricto + Supabase
(PostgreSQL + Auth + RLS) + Claude (claude-sonnet-4-6) + OpenAI Whisper
+ Google Vision + n8n (WhatsApp, solo orquestacion) + Vercel.

datos reales, pulido de demo y confirmar credenciales/migraciones de produccion.

2. CONTEXTO DE PRODUCTO
--------------------------------------------------------------------------------
Problema observado:
  - Muchos negocios pequenos venden todos los dias, pero no saben exactamente
    cuanto ganaron, cuanto gastaron o que producto les deja mas margen.
  - El registro manual tradicional les parece complejo o ajeno.
  - Las entidades financieras no ven un historial confiable, aunque el negocio
    tenga movimiento real y capacidad de pago.

Solucion propuesta:
  - Una app simple, mobile-first, con lenguaje de bodeguero.
  - Un asesor por WhatsApp que entiende frases naturales como:
      "vendi 3 arroces a 5 soles"
      "me deben 20 soles"
      "compre una caja de leche"
  - Un PymScore que resume salud financiera y puede convertirse en expediente
    para credito responsable.
  - Educacion financiera progresiva, desbloqueada por niveles.

Usuario objetivo:
  - Dueño de bodega, minimarket, puesto o comercio pequeño.
  - Usa WhatsApp todos los dias.
  - Puede no saber terminos como "margen", "flujo de caja" o "capital de trabajo".
  - Necesita ver "cuanto ganaste", "que se te esta acabando" y "quien te debe".

Decision de enfoque:
  Impulsa no compite como app contable pesada. La posicion es "copiloto
  financiero por WhatsApp": captura operaciones cotidianas y transforma ese
  historial en aprendizaje, alertas y evidencia financiera.

4. NAVEGACION (5 secciones del nav inferior)
--------------------------------------------------------------------------------
/inicio       Dashboard del dia: resumen, PymScore, accesos rapidos
/registrar    Tabs Foto / Voz / Manual + toggle Venta/Compra
/mi-negocio   Tabs Flujo / Inventario / Top / Fiado
/pym-score    Score, componentes y evolucion (nivel 2+)
/aprender     Modulos por nivel + examen

La navegacion esta optimizada para telefono. No hay sidebar ni interfaz de
escritorio compleja: el nav inferior contiene las 5 tareas centrales del dueño.
Cada pagina vive dentro de DashboardShell, que monta TopBar + NavInferior.

Detalle funcional por seccion:

  /inicio
    Vista de aterrizaje despues del login. Resume el dia, muestra PymScore,
    accesos rapidos y avisos accionables.

  /registrar
    Punto principal de captura. Tiene tabs Foto / Voz / Manual y permite elegir
    entre Venta y Compra. La idea es que el usuario registre en menos de un
    minuto sin saber de contabilidad.

  /mi-negocio
    Vista operativa: flujo de dinero, inventario, top productos y fiado. Es la
    seccion para responder "como va mi negocio hoy".

  /pym-score
    Salud financiera del negocio. Muestra score circular, componentes y
    evolucion. Es la base para explicar riesgos, mejoras y acceso a credito.

  /aprender
    Contenido educativo segun nivel. Permite reforzar conceptos y subir de
    nivel con modulos/examen.

Auth:        /login, /registro
Onboarding:  /onboarding (quiz + agente Claude que asigna nivel/XP)
Raiz /:      redirige a /login o /inicio segun sesion

Rutas legacy (redirigen automaticamente, codigo en app/(dashboard)/_legacy/):
  /ventas, /ventas/nueva       -> /registrar
  /inventario, /nueva-guia     -> /mi-negocio | /registrar
  /gastos                      -> /registrar
  /clientes                    -> /mi-negocio

5. ARQUITECTURA TECNICA
--------------------------------------------------------------------------------
Arquitectura general:

  Cliente PWA (Next.js App Router)
      ↓
  API Routes server-side
      ↓
  Supabase (Auth + PostgreSQL + RLS)
      ↓
  Servicios externos solo desde servidor:
      Claude, OpenAI Whisper, Google Vision, Twilio media

Arquitectura WhatsApp:

  Usuario WhatsApp
      ↓
  Twilio Sandbox / WhatsApp API
      ↓
  n8n webhook (solo normaliza payload)
      ↓
  POST /api/chatbot en Vercel
      ↓
  Next.js resuelve negocio, contexto, Claude, acciones y Supabase
      ↓
  n8n arma TwiML
      ↓
  Twilio envia respuesta

Regla clave:
  n8n NO consulta Supabase y NO llama a Claude. n8n es el cartero. La API de
  Next.js es el cerebro. Esto mantiene la logica de negocio versionada, testeable
  y protegida con service role solo en servidor.

Patron de seguridad:
  - API routes normales: sesion Supabase + negocio_id derivado del usuario.
  - /api/chatbot: header x-n8n-secret + telefono_wsp del negocio.
  - Nunca se acepta negocio_id desde body/query.
  - Service role solo en server.
  - RLS habilitado en tablas sensibles.

7. BASE DE DATOS (Supabase) — migraciones
--------------------------------------------------------------------------------
001_schema_inicial.sql ........ core: negocios, productos, ventas, items_venta,
                                 gastos, clientes, proveedores, niveles,
                                 pym_scores + RLS
002_registrar_venta_rpc.sql ... RPC atomica venta + descuento de stock
003_rediseno.sql .............. categorias, deuda proveedores, componentes
                                 PymScore, perfil IA
004_examen.sql ................ banco de preguntas + intentos + seed
005_conversaciones_wsp.sql .... estado del chatbot WhatsApp (estado_flujo,
                                 contexto, historial) + RLS
005_indices.sql ............... indices en gastos, clientes, proveedores,
                                 items_venta, guias_proveedor (auditoria DB-02)
006_examen_seguro.sql ......... revoca SELECT de respuesta_correcta a
                                 anon/authenticated (auditoria DB-01)
007_chatbot_wsp_memoria_idempotencia.sql
                               conversaciones_wsp (memoria) +
                               mensajes_wsp_procesados (idempotencia MessageSid)

Orden recomendado: 001 → 002 → 003 → 004 → 005_conversaciones_wsp →
005_indices → 006 → 007

Modelo de datos principal:
  - negocios: negocio del usuario y telefono_wsp para WhatsApp.
  - productos: inventario, stock minimo, compra/venta.
  - ventas + items_venta: registro transaccional y detalle de productos.
  - gastos: egresos categorizados.
  - clientes/proveedores: fiado, deudas y relaciones comerciales.
  - pym_scores: historial de salud financiera.
  - progreso_usuario/progreso_modulos: sistema de niveles y aprendizaje.
  - conversaciones_wsp: memoria del bot por telefono/negocio.
  - mensajes_wsp_procesados: idempotencia frente a reintentos de Twilio.

Puntos importantes:
  - La venta atomica usa RPC para crear venta, items y descontar stock.
  - Las tablas de niveles filtran por user_id, no por negocio_id.
  - conversaciones_wsp solo debe ser tocada por /api/chatbot.
  - La respuesta correcta del examen no queda expuesta al cliente gracias a
    privilegios por columna en 006_examen_seguro.sql.

12. DIRECCION DE PRODUCTO (propuesta de diferenciacion)
--------------------------------------------------------------------------------
No competir como app contable (clon de Treinta). Posicionar Impulsa como
"copiloto financiero por WhatsApp" que convierte el desorden diario en un
historial financiero (PymScore) para acceder a credito responsable.
Detalle completo en: docs/propuesta-innovacion-impulsa.docx

Narrativa para jurado:
  - El problema no es falta de esfuerzo del emprendedor, sino falta de historial
    ordenado y comprensible.
  - Impulsa aprovecha un canal que el usuario ya domina: WhatsApp.
  - El sistema no exige que el usuario aprenda contabilidad desde el dia uno;
    adapta el vocabulario y las funciones segun su nivel.
  - El PymScore traduce operaciones diarias en una señal financiera entendible.
  - Con suficiente historial, la app puede ser puente hacia credito mas justo.

Lo que NO se esta intentando resolver en esta version:
  - Contabilidad formal completa.
  - Facturacion electronica.
  - Paginacion/reporteria avanzada.
  - Integracion con Instagram u otros canales.
  - Motor crediticio regulado. El PymScore es una señal/expediente, no una
    aprobacion automatica de credito.
